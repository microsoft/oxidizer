# Routerama Performance Report

Generated by `scripts/perf_report.rs`:
- `cargo bench --bench criterion_routers` â€” criterion wall-clock timings.
- `cargo bench --bench gungraun_routers` â€” Callgrind instruction-precise counts.
- `cargo bench --bench routerama_query` â€” differential query-codec timings.
- `cargo bench --bench routerama_query_cg` â€” differential query-codec instruction counts.

**Workload:** one full sweep of the shared request-path lookups (see `benches/common/routes_data.rs`) against each router. Every router is built from the same route table (literal segments plus single-segment `{var}` parameters â€” the common subset all of them express) in a setup step that is excluded from the measured region. Setup also performs one full unmeasured lookup sweep to initialize lazy matcher and allocator state before measuring the steady-state hot path; `routerama_static` is the compile-time `#[resolver]` router, so it has no construction cost, while `routerama_dynamic` is the run-time router registered from the same table.

**Apples-to-apples:** every router is driven to the same *typed* end state â€” the HTTP method (verb) validated against the request and every captured path variable coerced into its declared type (`u32` parsed, `String` percent-decoded and owned, `&str` borrowed). `routerama` reaches this in one step (a typed enum variant with the method already matched and every field coerced); the third-party routers only *select* a route, so the harness explicitly checks the method and coerces each parameter the same way afterwards. `regex` selects the winner with a `RegexSet` and then re-scans it with the winning `Regex` to capture (two passes), so it does structurally more work â€” read it as an upper bound for a regex-based router reaching the same end state.  
Criterion median is reported (default 30 samples, 1 s warm-up, 2 s measurement; override with `--samples` / `--measurement-time` / `--warm-up-time`).  
Instructions, branch misses, and memory accesses are per full lookup sweep.  
Memory accesses = L1 Hits + LL Hits + RAM Hits (Callgrind D-cache references).
